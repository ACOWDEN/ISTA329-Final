How Does It Work?

How Blockchains Work

A blockchain is database network that securely records transactions across a bunch of different computers. Multiple copies of the blockchain system exist and run on different devices (nodes) that anyone can join in running themselves.

When a user makes a transaction, it is verified by every device in the network and waits to be added to the unchangeable history of transactions. When it's ready, the transaction, along with other transactions waiting to be added, is compiled into a "block" and added to the history of transactions (all the previous blocks). This creates a chronological history of transactions made on the network, proving the context of every user's current balance at any given time. Once a block is added, it cannot be changed or removed from the network.

Consensus is vital in this process. Agreement among all these devices (consensus) ensures accurate transaction records without a central controller. It ensures that all the copies of the ledger agree on the transactions and the order they're added. Each device running the blockchain software will only update their history of transactions if they agree that the update is valid.

This is how a database can exist not as a single instance, but a bunch of instances that are all the same and update “in unison”. In this, no single entity can change the blockchain database or remove data from it, as all other nodes in the network will not recognize this change as valid and therefore not make it to their own databases. Similarly, if anyone inserts an invalid transaction to be added to the blockchain, it will not be recognized by the rest of the network. Users make transactions on the blockchain using a blockchain wallet. A transaction may be sending crypto assets or interacting with some application.
